General information

This Discord bot has been developed over the past few months to be made up out of 3 languages for 3 bots. C# Discord.net - Which most of the bot has been wrote in and handles most features; Discord.py - Which the music, tts, some Minecraft and pagination systems have been wrote in and Discord.js - which the web UI and API has been wrote in. By default, the prefix is the ampersand symbol(&) but this can be configured depending on preference.

To get a list of commands, use the &help command, to get info such as syntax and a description of a command, use the &help command_name.

Prefix configuration

Note: This command reuqires permissions GuildPermissions.Administrator

&prefix new_prefix

This bot supports full custon guild prefixes for each individual guild, which can be modified at any given time with the prefix command.

To change the prefix, we use the default prefix being the ampersand symbol(&) and we use: &prefix new_prefix. This means that the next time you use that new_prefix, it'll be treated as the & was originally. For example: &prefix * sets the prefix to *, and doing *ping will execute the ping command.

Welcome channel configuration

Note: This command reuqires permissions GuildPermissions.ManageChannels

&welcomechannel desired_channel

When a user joins a guild, Discord allows you to send a standard message, but with FinBot you are able to configure a channel of your chosing to send a custom welcoming embed too, this can be changed at any time and also sends a message when a user leaves the guild.

This can be done with the welcomechannel command, which can be used in these ways:

    Note: this only works on text channels.
  • &welcomechannel channel_id
  • &welcomechannel channel_name
  • &welcomechannel #channel
  • &welcomechannel <#channel_id>
    •

This allows for users to easily configure their desired channel with ease whilst not having to conform to one standard syntax.

Enable/disable levelling

Note: This command reuqires permissions GuildPermissions.Administrator

&enablelevelling on_or_off

When it comes to servers, one of the most engaging features to have is a levelling system where users can fight for points to become the most active users, FinBot uses a complex levelling system with lots of functionality to get the most out of this! By default, this comes disabled, but it's easily enabled with the &enablelevelling command.

This command can be used as such:

To turn enable levelling:
  • &enablelevelling true
  • &enablelevelling on
    • To disable levelling:
  • &enablelevelling false
  • &enablelevelling off
    •

    This will either enable or disable levelling, which means that a user can earn between 15-25XP per message at one or more minute intervals(to avoid spam messages counting towards XP).

    Note: This command reuqires permissions GuildPermissions.ManageChannels

    &modchannel desired_channel>

    When a user gains themselves an infraction, it is logged to a database, but that's of no help for moderators wanting to keep on top of users infractions. This feature means that every infraction and some information about it can be sent into a dedicated channel of your liking.

      Note: this only works on text channels.
    • &modlogchannel channel_id
    • &modlogchannel channel_name
    • &modlogchannel #channel
    • &modlogchannel <#channel_id>
      •

    And all user-gained infractions will be posted there with details about the infraction.

    Message purging

    Note: This command reuqires permissions GuildPermissions.ManageMessages

    &clear purge_amount

    When working with servers, it's common for moderators to want to purge a large portion of messages, with the &clear command that makes it entirerly possible!

    When a purge_amount is set to under 15, it instantly purges the last number of messages. When a number over 15 is selected, it will ask for conformation, to which you can reply "yes" to confirm the purge of however many messages you would like to remove. This also does not clog up the snipe command.

    Enable and disable links

    Note: This command reuqires permissions GuildPermissions.ManageChannels

    &disablelinks on_or_off

    In community servers, it's not uncommon for moderator to not want users to send links. This is because there's an increased risk of a user sending a potentially malicious link that someone else might click on. You are able to toggle whether users can send links/Ip addresses/emails though this bot with one simple command with the following syntax:

      Disable users to send links
    • &disablelinks true
    • &disablelinks on
      • Enable users to send links
    • &disablelinks false
    • &disablelinks off

    Unlike most link-catchers, this bot catches URLs, IP addresses, emails and a combination of these together to make a strict anti-link system.

    Copy emotes

    Note: This command requires permissions GuildPermissions.ManageEmojis

    &copyemote emote_to_copy

    This bot offers a feature which allows you to "steal" or copy another guilds emote, you are able to do this to as many emojis as you want.

    You are able to do this with this syntax:

    • &copyemote emote_url
      • Note: This syntax only works if the bot is in that guild.
    • &copyemote emote_name emote_guild_id new_emote_name
